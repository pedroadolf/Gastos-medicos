## 🎨 Prompt para diseñador de UI/UX

```text
Diseña un dashboard para un seguro de Gastos Médicos Mayores (GMM) con las siguientes características:

## 🎨 Estilo visual general
- Fondo tipo "folder de expediente médico" o carpeta de cartulina (textura suave, color cálido neutro).
- Menú superior claramente separado del resto del contenido (con sombra o línea divisoria).
- Tarjetas flotantes con bordes suaves, sombras ligeras y fondo blanco o semitransparente.
- Inspiración en interfaz clínica/hospitalaria pero moderna y limpia.

## 🎨 Paleta de colores obligatoria
- #FFFFFF (blanco, fondos de tarjetas)
- #FFAA00 (amarillo, acentos, alertas medias)
- #343434 (gris oscuro, textos principales)
- #D8D9D7 (gris claro, bordes, fondos secundarios)
- #B22B21 (rojo, solo para cosas muy relevantes: alertas críticas, rechazos, consumos altos)

## 🌓 Modo oscuro/claro
- Debe incluir un toggle para cambiar entre modo claro y modo oscuro.
- En modo oscuro, los colores se adaptan manteniendo la jerarquía visual.

## 📱 Menú superior (separado del contenido)
Las opciones del menú son:
- Dashboard
- Nuevo Trámite
- Mis Trámites (Historial)
- Observabilidad
- Configuración

## 📊 Contenido del Dashboard (obligatorio)

### 1. Vista Global (Póliza general)
Mostrar:
- Suma Asegurada Total (ej. $5,000,000 MXN)
- Suma Consumida (ej. $1,200,000 MXN)
- Suma Disponible (ej. $3,800,000 MXN) con semáforo verde/amarillo/rojo.
- Gráfico de dona/torta que muestre distribución del consumo entre los 4 asegurados.

### 2. Vista Individual por Asegurado (4 tarjetas)
Cada tarjeta debe mostrar:
- Nombre, Edad, Parentesco (Titular, Esposo, Hijo 1, Hija 2)
- Padecimiento principal activo (Tumor Cerebral, Reemplazo de Rodilla, Asma Severo, Sin siniestros)
- Barra de progreso de Sub-límite por padecimiento (ej. "Sub-límite Oncología: $750k / $1M")
- Estado del deducible (Cumplido / En proceso / N/A)
- Número de siniestros abiertos
- Botón "Ver detalle"

### 3. Tubería de Siniestros (Kanban o tabla)
Columnas:
- En Trámite / Revisión
- Pre-autorizados
- En Pago
- Rechazados / Observaciones (usar color #B22B21 para esta sección)

### 4. Desglose Financiero por Asegurado
- Gráfico de barras apiladas: 4 barras (una por asegurado), divididas por tipo de gasto (Hospitalización, Medicamentos, Estudios, Honorarios).
- Tabla de "Gasto de Bolsillo" (lo que el asegurado ha pagado por su cuenta).

### 5. Centro de Alertas y Próximos Pasos
Mostrar alertas con íconos:
- 🔴 Roja (#B22B21): consumo de sub-límite alto o rechazos
- 🟡 Amarilla (#FFAA00): vencimientos próximos
- 🟢 Verde: aprobaciones recientes
- 📅 Calendario de próximas citas/cirugías pre-autorizadas

## ⚠️ Restricción técnica importante
El rediseño es SOLO VISUAL. No se deben cambiar nombres de variables, IDs, endpoints de API, ni la lógica de comunicación con n8n o Supabase. El backend debe seguir funcionando exactamente igual.

## 📏 Formato de entrega esperado
- Diseño en Figma (o código HTML/CSS/JS responsive)
- Vista mobile y desktop
- Componentes reutilizables
- Toggle claro/oscuro funcional (si es código)
```

## 5. ⚙️ JavaScript para modo oscuro (`js/dark-mode.js`)

```js
const toggle = document.getElementById('darkModeToggle');
const body = document.body;

toggle.addEventListener('click', () => {
    body.classList.toggle('dark-mode');
    const isDark = body.classList.contains('dark-mode');
    toggle.textContent = isDark ? '☀️ Modo claro' : '🌙 Modo oscuro';
});
```

## ✅ Lo que el diseñador puede modificar sin riesgo

| Sí puede | No puede |
|----------|----------|
| Colores, fuentes, sombras, bordes | Cambiar `id`, `class` usadas en JS |
| Espaciados, tamaños, posiciones | Modificar `data-*` attributes |
| Fondo tipo folder (textura, opacidad) | Renombrar botones de acción |
| Animaciones, transiciones | Cambiar estructura de tarjetas |
| Layout responsivo | Tocar nombres de secciones críticas |

## 3. 🌙 JavaScript para modo oscuro (`js/dark-mode.js`)

```js
const toggle = document.getElementById('darkModeToggle');
const body = document.body;

// Revisar preferencia guardada
if (localStorage.getItem('darkMode') === 'true') {
    body.classList.add('dark');
    toggle.textContent = '☀️ Modo claro';
}

toggle.addEventListener('click', () => {
    body.classList.toggle('dark');
    const isDark = body.classList.contains('dark');
    localStorage.setItem('darkMode', isDark);
    toggle.textContent = isDark ? '☀️ Modo claro' : '🌙 Modo oscuro';
});
```

## ✅ Resumen de lo que el diseñador puede modificar SIN riesgo

| ✅ Puede cambiar | ❌ No puede cambiar |
|----------------|---------------------|
| Colores (`bg-*`, `text-*`, `border-*`) | `id` de elementos críticos |
| Tipografías (clases de Tailwind) | `data-*` attributes |
| Sombras, bordes, redondeados | Nombres de botones con eventos |
| Espaciados (`p-*`, `m-*`, `gap-*`) | Estructura de tarjetas (grid) |
| Fondo tipo folder (en `custom.css`) | URLs de APIs o webhooks |
| Modo oscuro (colores en `custom.css`) | Clases usadas por JS externo |
